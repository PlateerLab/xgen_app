### 2. InferenceEngine 구현

**파일**: `src-tauri/src/services/inference.rs`

#### 2.1 모델 로딩 (`load_model`)
```rust
pub async fn load_model(&mut self, config: ModelConfig) -> Result<ModelStatus>
```

- `GgufModelBuilder`를 사용한 GGUF 모델 로딩
- `best_device()`로 자동 GPU/CPU 선택 (CUDA/Metal/CPU)
- 옵션 지원:
  - `tokenizer_id`: HuggingFace 토크나이저 ID
  - `chat_template`: 커스텀 채팅 템플릿
  - `paged_attention`: 긴 컨텍스트용 Paged Attention

#### 2.2 텍스트 생성 (`generate`)
```rust
pub async fn generate(&self, request: GenerateRequest) -> Result<GenerateResponse>
```

- `TextMessages`로 시스템 프롬프트 + 사용자 프롬프트 구성
- `send_chat_request()`로 비스트리밍 생성
- 응답 통계: `prompt_tokens`, `completion_tokens`, `generation_time_ms`, `tokens_per_second`

#### 2.3 스트리밍 생성 (`generate_stream_with_callback`)
```rust
pub async fn generate_stream_with_callback<F>(
    &self,
    request: GenerateRequest,
    mut callback: F,
) -> Result<GenerateResponse>
where
    F: FnMut(String) + Send,
```

- mistral.rs의 커스텀 `Stream` 타입이 `futures::Stream` 트레이트를 구현하지 않아 콜백 방식으로 구현
- `stream.next().await`로 각 토큰 수신
- `Response::Chunk`, `Response::Done`, 에러 타입 처리

## 해결한 이슈

### 1. objc2 버전 충돌
- **문제**: Tauri와 mistralrs 간 objc2 버전 충돌
- **해결**: `Cargo.lock` 삭제 후 재빌드

### 2. Model 타입 처리
- **문제**: `Model`이 trait이 아닌 struct
- **해결**: `Model as MistralModel` 별칭 사용

### 3. Stream 타입 비호환
- **문제**: mistral.rs의 `Stream`이 `futures::Stream` 미구현
- **해결**: 콜백 기반 `generate_stream_with_callback` 함수로 대체

### 4. MCP 클라이언트 통합 (`mistralrs_mcp`)

#### 4.1 McpConfigManager 업데이트

**파일**: `src-tauri/src/services/mcp_manager.rs`

```rust
use mistralrs_mcp::{
    McpClientConfig as MistralMcpClientConfig,
    McpServerConfig as MistralMcpServerConfig,
    McpServerSource,
};

impl McpConfigManager {
    /// Convert enabled servers to mistralrs_mcp McpClientConfig
    pub fn to_mcp_client_config(&self) -> Option<MistralMcpClientConfig>
}
```

- `McpServerConfig` → `MistralMcpServerConfig` 변환
- `McpConnectionType::Stdio` → `McpServerSource::Process`
- `McpConnectionType::Http` → `McpServerSource::Http`
- 활성화된 서버만 포함

#### 4.2 InferenceEngine MCP 지원 추가

**파일**: `src-tauri/src/services/inference.rs`

```rust
pub struct InferenceEngine {
    // ... 기존 필드
    mcp_client: Option<McpClient>,
    mcp_tool_count: usize,
}

impl InferenceEngine {
    /// Load a GGUF model with MCP tool support
    pub async fn load_model_with_mcp(
        &mut self,
        config: ModelConfig,
        mcp_config: McpClientConfig,
    ) -> Result<ModelStatus>
}
```

- `McpClient` 초기화 및 MCP 서버 연결
- `get_tool_callbacks_with_tools()`로 도구 콜백 획득
- `GgufModelBuilder.with_tool_callback_and_tool()`로 도구 등록
- 모델 언로드 시 MCP 클라이언트도 정리

#### MCP 통합 흐름

```
McpConfigManager                InferenceEngine
     │                                │
     │ to_mcp_client_config()         │
     ├───────────────────────────────►│
     │    MistralMcpClientConfig      │
     │                                │
     │                         McpClient::new()
     │                         McpClient::initialize()
     │                         get_tool_callbacks_with_tools()
     │                                │
     │                         GgufModelBuilder
     │                           .with_tool_callback_and_tool()
     │                           .build()
     │                                │
     │                         Model with MCP Tools
```

### 5. Frontend Tauri IPC 래퍼 작성

**경로**: `src/lib/tauri/`

#### 생성된 파일 구조

```
src/lib/tauri/
├── index.ts      # 메인 export
├── types.ts      # TypeScript 타입 정의
├── gpu.ts        # 하드웨어 정보 (getHardwareInfo)
├── model.ts      # 모델 관리 (listModels, downloadModel, deleteModel)
├── llm.ts        # LLM 추론 (loadModel, generate, embedText)
├── mcp.ts        # MCP 서버 관리 (addMcpServer, setMcpServerEnabled)
└── mode.ts       # 앱 모드 (setAppMode, checkGatewayConnection)
```

#### 주요 기능

##### 타입 정의 (`types.ts`)
- Rust 구조체와 1:1 매핑되는 TypeScript 인터페이스
- `ModelStatus`, `HardwareStatus`, `McpServerStatus` 등

##### LLM 추론 (`llm.ts`)
```typescript
// 스트리밍 생성
await llm.generate('Hello!', {
  maxTokens: 256,
  onToken: (token) => console.log(token),
  onDone: (stats) => console.log(`${stats.tokensPerSecond} tok/s`),
});
```
- `llm:token`, `llm:done` 이벤트 자동 구독
- cleanup 함수 반환으로 리스너 정리

##### MCP 서버 관리 (`mcp.ts`)
```typescript
await mcp.addMcpServer({
  name: 'filesystem',
  connectionType: 'stdio',
  command: 'npx',
  args: ['-y', '@modelcontextprotocol/server-filesystem', '.'],
});
```

#### 사용 예시

```typescript
import { llm, model, mcp, gpu } from '@/lib/tauri';

// 하드웨어 확인
const hw = await gpu.getHardwareInfo();
console.log(`Backend: ${hw.recommendedBackend}`);

// 모델 로드
const status = await llm.loadModel({
  modelPath: '/path/to/llama.gguf',
  contextLength: 8192,
});

// 텍스트 생성
await llm.generate('Hello!', {
  onToken: (t) => process.stdout.write(t),
});
```
